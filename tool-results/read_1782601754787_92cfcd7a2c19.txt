  1873→=== BUSGO ARCHITECTURE MAP ===
  1874→
  1875→BUSGO is a sub-application of SmarticketS for bus companies. It provides:
  1876→- Agent dashboard (trajets, embarquement, scanner, guichet, billets, incidents, equipe, voix, notifications)
  1877→- Passenger PWA at `/pwa-passager/` (separate from /busgo)
  1878→- Controller PWA at `/pwa-controleur`
  1879→- Service worker at `/sw-busgo-passenger.js`
  1880→
  1881→The BUSGO flow:
  1882→1. Agent creates a departure (trajet) via `/busgo/trajets` → POST `/api/busgo/trajets`
  1883→2. Agent sells a ticket at the guichet via `/busgo/guichet` → POST `/api/busgo/guichet/sell` (generates QR code with ticket info)
  1884→3. Passenger scans the QR code → opens `/pwa-passager/install?data=${base64}` → installs PWA + receives welcome notification
  1885→4. Passenger opens PWA dashboard at `/pwa-passager` → sees ticket, chronometer, agent contact
  1886→5. Agent starts boarding via `/busgo/embarquement/[departureId]` → displays agent QR code (encodes `/pwa-passager/scan?dep=${departureId}`)
  1887→6. Passenger scans agent QR at `/pwa-passager/scan` → POST `/api/busgo/embarquement/scan` → marks ticket as BOARDED
  1888→7. Agent can mark passengers late, signal delays, etc.
